## 2. WBS Dictionary Principles

- Structure is **deliverable-oriented** and aligned to scope baseline.
- Level 1 = Program, Level 2 = Major Deliverables, Level 3+ = Work Packages.
- Each work package has a clear owner, acceptance criteria, and measurable output.

## 6. WBS Coding Standards

- Use numeric hierarchical coding: `x.x.x`.
- Each schedule activity and cost line item must map to one WBS code.
- All risks and changes must reference impacted WBS element(s).

## 7. Baseline Control Rules

- WBS baseline changes require approved Change Control Board request.
- New work packages must include updated scope statement, budget line, and schedule impact.
- Deleted work packages require formal confirmation of out-of-scope disposition.
