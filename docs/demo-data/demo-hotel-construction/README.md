# Demo Input Pack - Hotel Construction Project

This folder contains a complete set of **English project management documents** for a large hotel construction program.

The package is designed for two purposes:

1. **Application testing** (document ingestion, extraction, search, and reporting)
2. **Demo data generation** (realistic PM artifacts with linked content)

## Project Scenario

- **Project Name:** Azure Bay Hotel & Convention Center
- **Location:** La Goulette, Tunis, Tunisia
- **Project Type:** New build, upscale urban hotel
- **Capacity:** 220 rooms + conference center + wellness facilities
- **Budget Baseline:** USD 86,000,000
- **Planned Duration:** 26 months (Jan 2026 to Mar 2028)

## Document Index

1. `01-project-charter.md` - Formal authorization and strategic objectives
2. `02-stakeholder-register.md` - Stakeholder mapping and engagement model
3. `03-work-breakdown-structure.md` - Deliverable hierarchy and acceptance criteria
4. `04-master-schedule.md` - Baseline milestones and critical path summary
5. `05-budget-baseline.md` - CAPEX baseline, cashflow, and control thresholds
6. `06-risk-register.md` - Quantified risks and response plans
7. `07-raid-log.md` - Consolidated Risks, Assumptions, Issues, Dependencies
8. `08-change-control-log.md` - Change request pipeline and impact history
9. `09-communication-plan.md` - Governance ceremonies and communication matrix
10. `10-weekly-status-report-sample.md` - Executed sample status report
11. `11-app-test-data-scenarios.md` - Structured data set for QA and demo walkthroughs
12. `12-procurement-plan.md` - Contracting strategy and package-level sourcing plan

## Suggested Test Uses

- Validate extraction of: IDs, dates, costs, owners, statuses, priorities
- Validate cross-document references (e.g., `R-07` in Risk Register and RAID)
- Validate trend analytics with schedule and cost variances
- Validate RAG filtering by workstream, severity, and owner
- Validate dashboard widgets (milestones, top risks, open changes)
