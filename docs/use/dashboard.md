## Purpose

The **Dashboard** is the operational landing page for each user. It combines a personal context summary (current project + user identity), quick navigation cards, and a recent activity feed.

![Dashboard overview](../../static/img/screenshots/01-dashboard.png)

## Why this matters

Dashboards reduce time-to-orientation. Instead of manually opening multiple screens, users can immediately see what is active, what is missing (for example no project selected), and where to act next.

## Who can use it

- **All signed-in users**

## Before you begin

- Sign in.
- For project-scoped metrics, make sure a project is selected in the top **Project Switcher**.

## Steps

1. Open **Dashboard** from the left navigation.
2. Confirm the page is loaded (title **Dashboard**).
3. Check the **Current project** card:
   - If it shows **Pick a project**, open **Projects** and choose one.
   - If a project is already selected, continue.
4. Review key cards:
   - **Projects**
   - **Knowledge**
   - **Marketplace**
   - **Current project**
5. Review **Recent projects** and use one-click links to jump to workspaces.
6. If needed, open **Portfolio Command Center** for a cross-project snapshot.

## Expected results

- Dashboard loads without errors.
- Quick cards and links are actionable.
- User can navigate to Projects/Knowledge/Marketplace in one click.

## Common issues

- **Current project shows “Pick a project”**: no project is selected yet; choose one in **Projects**.
- **Counts or project details look incomplete**: refresh after selecting a project.

## Tips

- Use Dashboard as a daily start point before entering Workspace chat or PM Docs.
- Keep one project selected while working to ensure PM Docs, Knowledge, and AI Log stay context-aligned.
