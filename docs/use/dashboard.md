## Purpose

The **Dashboard** is the operational landing page for each user. It combines current project context, readiness signals, quick navigation, and recent activity so you can decide what to do next without opening multiple pages first.

![Dashboard overview](../../static/img/screenshots/01-dashboard.png)

## Why this matters

Dashboards reduce time-to-orientation. Instead of manually opening multiple screens, users can immediately see what is active, what is missing (for example no project selected), and where to act next.

## Who can use it

- **All signed-in users**

## Before you begin

- Sign in.
- For project-scoped metrics, make sure a project is selected in the top **Project Switcher**.

## What you see on the Dashboard

- **Hero summary** with the main call to action for your current state
- **Project summary card** showing the active project and signed-in user
- **Quick stats** for projects, knowledge, actions, and project context
- **Command center** readiness panels when a project is selected
- **Recommended next steps** based on project conditions
- **Recent activity** so you can jump from orientation into action

If no project is active, the Dashboard switches into a guided onboarding state with clear links to **Projects**, **Portfolio Command Center**, and **AI Log**.

## Steps

1. Open **Dashboard** from the left navigation.
2. Confirm the page is loaded (title **Dashboard**).
3. Check the shell and hero area for your current state:
   - If the Dashboard shows **Pick a project** or **All projects**, open **Projects** or use the **Project Switcher**.
   - If a project is already selected, continue with the project-scoped readiness view.
4. Use the hero action buttons:
   - **Open workspace** when you want to continue execution work in the current project
   - **AI Log** when you want traceability for recent runs and activity
   - When no project is active, use **Go to Projects** to pick the correct context first
5. Review the summary cards and readiness sections:
   - **Delivery readiness**
   - **Execution momentum**
   - **Governance coverage**
   - **Knowledge depth**
6. Review **Recommended next steps** to see what the product suggests you do next.
7. Use **Recent projects** to reopen a project quickly when no project is active.
8. Use **Recent activity** to move into **AI Log** when you need detail.

## Expected results

- Dashboard loads without errors.
- Quick cards and links are actionable.
- You can tell immediately whether you are in a project-scoped view or a global no-project state.
- You can navigate to the next likely action in one click.

## Common issues

- **Current project shows “Pick a project”**: no project is selected yet; choose one in **Projects**.
- **Counts or project details look incomplete**: refresh after selecting a project.
- **Knowledge, Agents, PM Docs, or AI Log opens in the wrong context**: switch projects from the top bar and then retry navigation so project-aware links are regenerated with the correct project.
- **Marketplace is missing from the shell**: this is expected unless your role includes Marketplace access.

## Tips

- Use Dashboard as a daily start point before entering Workspace chat or PM Docs.
- Keep one project selected while working to ensure PM Docs, Knowledge, and AI Log stay context-aligned.
- If you are on mobile, open the sidebar menu first, then return to Dashboard when you need to reorient yourself.
