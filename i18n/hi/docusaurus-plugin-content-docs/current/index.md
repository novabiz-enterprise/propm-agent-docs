> **पढ़ना अत्यधिक अनुशंसित**
>
> प्रोपीएम एजेंट को कंपनी में अपनाने और एप्लिकेशन का पूर्ण लाभ उठाने के लिए, यह मार्गदर्शिका परियोजना की शुरुआत से ही पढ़ना और तैनाती, कनेक्टर सेटअप, शासन और दैनिक संचालन के दौरान संदर्भित करना अत्यंत अनुशंसित है।

यह दस्तावेज़ केवल उन फ़ंक्शनों का वर्णन करता है जो एप्लिकेशन, कोड, तैनाती कॉन्फ़िगरेशन और प्रोपीएम एजेंट के स्वचालित परीक्षणों में देखे गए हैं।

यह दो दर्शकों के लिए व्यवस्थित है :

- **व्यावसायिक उपयोगकर्ता** जो परियोजना में काम करते हैं ;
- **तकनीकी प्रशासक** जो पहुँच, एकीकरण, एआई प्रदाता, सदस्यता और निगरानी का प्रबंधन करते हैं।

## अनुशंसित एंड-टू-एंड मार्ग

वास्तविक उपयोग में, सबसे सामान्य प्रवाह निम्नलिखित है :

1. प्रकाशित URL तैनात या प्राप्त करें ;
2. Microsoft Entra ID खाते से लॉगिन करें ;
3. एक **सक्रिय परियोजना** बनाएं या चुनें ;
4. **परियोजना निर्माता** को प्रारंभिक उपयोगी भूमिकाएँ सौंपने दें ;
5. परियोजना **ज्ञान** लोड करें ;
6. **एजेंट** में एक संवाद प्रारंभ करें ;
7. **संरचित आउटपुट**, उसके साक्ष्य, ताजगी और विश्वास को पुनः पढ़ें ;
8. परिणाम को **कलाकृति** या **PM दस्तावेज़** में परिवर्तित करें, फिर **DOCX / XLSX** डाउनलोड, प्रकाशन या ज्ञान में जोड़ने से पहले समीक्षा करें ;
9. **प्रोएक्टिव सिग्नल**, **क्रियाएँ & अनुमोदन** और आवश्यकता पड़ने पर बाहरी एकीकरण प्रवाह को संभालें ;
10. केवल तब प्लेटफ़ॉर्म स्तर पर प्रबंधन करें जब विषय परियोजना से परे हो।

## लेबल सम्मेलन

जब प्लेटफ़ॉर्म फ्रेंच अनुवाद प्रदर्शित करता है, यह दस्तावेज़ निम्न लेबल को प्राथमिकता देता है :

- **कार्य क्षेत्र** ;
- **ज्ञान** ;
- **PM दस्तावेज़** ;
- **एआई लॉग** ;
- **प्रवेश नियंत्रण** ;
- **शासन नीतियाँ** ;
- **परियोजना एकीकरण** ;
- **प्लेटफ़ॉर्म एकीकरण** ;
- **एआई प्रदाता सेटिंग्स** ;
- **क्रियाएँ & अनुमोदन**।

जब एक अंग्रेज़ी लेबल अस्पष्टता दूर करने के लिए उपयोगी हो, तो उसे कोड में एक बार उद्धृत किया जाता है, उदाहरण के लिए `Trace ID` या `OpenAI-compatible`.

## प्रोपीएम एजेंट क्या कवर करता है

जांचित व्यवहारों से, प्रोपीएम एजेंट कार्य को **परियोजना संदर्भ** के आसपास व्यवस्थित करता है :

1. आप एक परियोजना खोलते या बनाते हैं ;
2. आप **ज्ञान** को पोषण देते हैं ;
3. आप **एजेंट** से पूछते हैं ;
4. आप **संरचित आउटपुट** पुनः पढ़ते हैं ;
5. आप इन परिणामों को **PM दस्तावेज़** और कलाकृतियों में परिवर्तित करते हैं ;
6. आप **ट्रेसबिलिटी** को **एआई लॉग** में सत्यापित करते हैं ;
7. आप **सिग्नल**, **डाइजेस्ट**, **ड्राफ्ट** और **शासनित क्रियाएँ** को संभालते हैं ;
8. आप केवल तभी **कनेक्टर्स** और **एकीकरण** का उपयोग करते हैं जब वे मान्य और परियोजना से बंधे हों ;
9. प्रशासक **प्लेटफ़ॉर्म प्रशासन** में तकनीकी परत को संचालित करते हैं।

## एप्लिकेशन को क्या अलग बनाता है

निम्नलिखित तत्व स्पष्ट रूप से इंटरफ़ेस, सीड डेटा या परीक्षणों में दिखाई देते हैं :

- **संरचित आउटपुट** जिनमें अनुभाग, साक्ष्य, ताजगी, विश्वास और निर्णय लेने के लिए अगले कदम होते हैं ;
- **रेखा** एक रन, संरचित आउटपुट, कलाकृति, कलाकृति संस्करण और PM दस्तावेज़ के बीच ;
- **प्रोएक्टिव सिग्नल** विरोधाभास, ताजगी, अवरोध, पुनः प्रयास और कुछ पोर्टफोलियो दबाव के लिए ;
- **शासनित दस्तावेज़ श्रृंखला** एक एजेंट परिणाम से **DOCX / XLSX** डाउनलोड, प्रकाशन और ज्ञान में पुनः इंजेक्शन तक ;
- **शासनित कनेक्टर्स** **प्लेटफ़ॉर्म एकीकरण**, **परियोजना एकीकरण** और **क्रियाएँ & अनुमोदन** के बीच अलग ;
- **एआई रनटाइम पारदर्शिता** वास्तविक प्रदाता, तैनाती पर चयनित प्रदाता और मॉडल परिवार के साथ।

## स्क्रीनशॉट और आरेख

एप्लिकेशन के स्क्रीनशॉट उस पर्यावरण में उत्पन्न किए गए हैं जिसे देखा गया था। जब कुछ उपलब्ध स्क्रीनशॉट केवल लोडिंग स्थिति या अपूर्ण डेमो डेटा दिखाते थे, तो इस फ्रेंच संस्करण ने भी **स्पष्टीकरण आरेख** शामिल किए हैं ताकि अनुवाद और अंतिम प्रकाशन से पहले उपयोगी पढ़ने को बनाए रखा जा सके।
